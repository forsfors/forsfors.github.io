Apache Dubbo 服务引用源码分析

发表于 2019-05-25 | 分类于 dubbo

Dubbo服务引用部分涉及consumer的注册，provider服务的发现，负载均衡，熔断，远程服务调用等功能。
本文以2.7.1中的dubbo-demo/dubbo-demo-api为例，注册中心改成了zookeeper://localhost:2181。本文要求至少理解dubbo的SPI机制。
服务引用大概流程：

  1. 注册consumer到zk
  2. 增加zk的监听，并对zk的数据（除consumer外）进行一次同步
  3. 生成Invoker，经包装后生成Invoker的代理
  4. 用户通过invoker发起接口调用，invoker通过负载均载，过滤，后发起tcp请求，得到结果后返回。

本文以代码块分类，并没有按上述流程分类。
